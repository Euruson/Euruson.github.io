Pelican and voce

在使用Pelican搭建自己的博客网站时，遇到了非常多的问题。网上关于Pelican的中文教程很少，有些问题和配置在说明文档里也是一带而过（可能是我太菜了吧）。这里列举一些问题并提供一些解决方案供。

关于Fabric脚本

因为我是在Win10环境下配置的，要用make的话有点麻烦，于是选择使用Fabric。

  1. 我用的是Python 3.6.1，旧版的Fabric不兼容，所以无法使用。
  2. 调用pip install fabric3 安装Fabric3。
  3. 安装好后调用far serve出错，发现没有SocketServer模块，python3中应该是socketserver,改掉即可。

关于voce主题

哇，第一眼就被这个简洁的风格吸引了，加上原博主的那个二位码的logo，感觉不能再好了。之后去github上看了下，并没有很火的样子。嗯，小众，典雅，就决定是你了。然而原来的创作者很久没维护了，落下了很多坑。

  1. Assets插件版本过旧 使用pip install pelican安装的是Pelican(3.7.1), 原来repo中的插件太老了不支持，会出现CRITICAL: KeyError: u'JINJA_EXTENSIONS'的问题， 下载getpelican/pelican-plugins/assets覆盖掉原来插件就行了。或者直接克隆我的repo中的voce。

  2. pelicanconf.py中的LINKS和SOCIAL 因为voce主题并没有menu，所以一些连接得放在LINKS中。SOCIAL因为有定制化的图标，所以只支持一些主流的社交或工作平台。(或许只是我没有发现使用的方法)

  3. TAGLINE 原创作者给的示范网站的logo下面有Tagline的索引，很漂亮。但是我一开始使用的时候那个地方只有一片丑陋的空白。这个地方真是让我废了一番功夫，先是把Pelican的文档基本读了一边，没找到办法。因为voce是从pelican-subtle和pelican-svbhack修改而来的，于是我又去这两个repo找了一下，发现有个TAGLINE的参数。但是使用了以后偶尔会生效，感觉没找到正确的方法。最后直接到templates/base.html中去看模板，发现必须到pelicanconf.py中设置TAGS_URL和ARCHIVES_URL两个参数。

  4. 我的pelicanconf.py 因为我发现大多数的问题都是因为没有给出配置文件，所以我把我的配置文件贴出来，其中也有一些其他的在这个过程中学到的东西。

#!/usr/bin/env python
# -*- coding: utf-8 -*- #
from __future__ import unicode_literals

#Author
AUTHOR = 'Euruson'
SITENAME = "Euruson's Blog"
SITEURL = 'http://localhost:8000'

#Markdown extensions
MARKDOWN = {
    'extension_configs': {
        'markdown.extensions.codehilite': {'css_class': 'highlight','linenums':'True'},
        'markdown.extensions.extra': {},
        'markdown.extensions.meta': {},
    },
    'output_format': 'html5',
}

#Content
PATH = 'content'
STATIC_PATHS = ['pages','images']
ARTICLE_PATHS = ['articles']
ARTICLE_SAVE_AS = '{date:%Y}/{date:%m}/{slug}.html'
ARTICLE_URL = '{date:%Y}/{date:%m}/{slug}.html'

DEFAULT_LANG = 'Chinese (Simplified)'

#Article settings & URL
DEFAULT_CATEGORY = 'Untitled'
TAGS_URL = 'tags.html'
ARCHIVES_URL = 'archives.html'


# Feed generation is usually not desired when developing
FEED_ALL_ATOM = None
CATEGORY_FEED_ATOM = None
TRANSLATION_FEED_ATOM = None
AUTHOR_FEED_ATOM = None
AUTHOR_FEED_RSS = None

# Blogroll
LINKS = (('Home','index.html'),
     ('About','pages/about.html'),)

# Social widget
SOCIAL = (('Feed','/feeds/all.atom.xml'),
      ('Email','mailto:xxx@qq.com'),
      ('GitHub','http://github.com/euruson'),)

DEFAULT_PAGINATION = 10

# Uncomment following line if you want document-relative URLs when developing
#RELATIVE_URLS = True

PLUGIN_PATHS = ["pelican-plugins"]
PLUGINS = ["assets"]

#Theme & its features
THEME = 'pelican-themes/voce'
MANGLE_EMAIL = True
USER_LOGO_URL = 'images/me.jpg'

关于Pelican

Pelican的Markdown或者说Python-Markdown是不支持Github Flavored Markdown的，必须使用Markdown的扩展extensions。

The Python-Markdown library (which Pelican uses to parse markdown documents) includes support for a number of built-in and third party extensions and many of those extensions offer support for configuration options to alter their behavior. As of Pelican 3.7, to choose the available Markdown extensions and configure the extensions that you are using, you can edit the MARKDOWN setting (cf. settings). Default options are:

MARKDOWN = {
    'extension_configs': {
        'markdown.extensions.codehilite': {'css_class': 'highlight'},
        'markdown.extensions.extra': {},
        'markdown.extensions.meta': {},
    },
    'output_format': 'html5',
}

In versions before 3.7, you can edit the MD_EXTENSIONS setting (cf. settings). These earlier versions do not offer a setting that would get passed into Markdown's extension_configs keyword, but Markdown does allow instances of extension classes to be passing in rather than the string names of the extensions. Hence, in these earlier versions, as Pelican's setting file is just Python, you can import your extensions, create an instance with the config options you want, and pass that in using Pelican's setting.

最坑爹的是，Python-Markdown不支持子结构的Fenced Block, Python Markdown v2.6.8 documentation：

Warning: Fenced Code Blocks are only supported at the document root level. Therefore, they cannot be nested inside lists or blockquotes.
